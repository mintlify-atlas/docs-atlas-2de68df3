This page lists all supported model types for Speech-to-Text (STT) and Text-to-Speech (TTS), including required files and download links.

## Speech-to-Text (STT) Models

<Note>
For **real-time (streaming) recognition** from a microphone or audio stream, use streaming-capable model types: `transducer`, `paraformer`, `zipformer2_ctc`, `nemo_ctc`, or `tone_ctc`.

See the [Streaming STT](/api-reference/stt-streaming) documentation for details.
</Note>

## Model Quantization

<Info>
The SDK automatically detects and prefers quantized (int8) models when available. For example, if both `model.onnx` and `model.int8.onnx` exist, the library chooses according to the `preferInt8` option in init.
</Info>

## Auto-Detection

The library detects model types from the **files present** in each model directory. Folder and file names do not need to follow any fixed convention.

To use auto-detection:

```typescript
const stt = await createSTT({
  modelPath: { type: 'asset', path: 'models/your-model-folder' },
  modelType: 'auto', // Auto-detect from files
});
```
